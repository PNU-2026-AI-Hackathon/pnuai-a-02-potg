# 모이라: 모두가 이어지는 라이브러리

> 작은도서관과 지역 주민, 사서, 대학생 봉사자를 연결하는 AI 기반 지역 커뮤니티 플랫폼

## 1. 프로젝트 소개

**모이라: 모두가 이어지는 라이브러리**는 작은도서관이 지역사회 소통과 문제 해결의 거점 공간으로 기능할 수 있도록 지원하는 AI 기반 커뮤니티 플랫폼입니다.

주민이 제안한 지역 의제를 AI가 분석·분류하고, 이를 바탕으로 사서의 프로그램 기획을 지원합니다. 또한 대학생 봉사자 매칭 기능을 통해 주민, 사서, 봉사자가 연결되는 지역밀착형 서비스를 제공합니다.

---

### 1.1. 개발배경 및 필요성

작은도서관은 지역 주민의 문화·교육 활동과 커뮤니티 형성을 지원하는 생활밀착형 문화공간입니다. 단순한 도서 대출 공간을 넘어 독서 활동, 문화 프로그램, 정보활용교육 등 다양한 서비스를 제공하며 지역 공동체 형성에 기여하고 있습니다.

그러나 실제 운영 과정에서는 다음과 같은 문제가 존재합니다.

- 작은도서관에 대한 지역 주민의 인식 부족
- 낮은 이용률과 부족한 홍보
- 제한적인 프로그램 운영
- 사서 및 운영 인력 부족
- 지역 단위 통합 플랫폼 부재
- 주민 수요를 반영한 프로그램 기획의 어려움

특히 작은도서관은 소규모 인력으로 운영되는 경우가 많아 사서가 행정 업무, 프로그램 기획, 민원 응대, 정보서비스 운영 등을 동시에 담당해야 하는 부담이 있습니다. 또한 주민들은 집 근처 작은도서관에서 어떤 프로그램이 운영되는지, 어떤 참여 기회가 있는지 쉽게 확인하기 어렵습니다.

이러한 문제는 작은도서관의 지역 커뮤니티 기능 약화와 문화·정보 접근성 격차로 이어질 수 있습니다. 이에 따라 본 프로젝트는 작은도서관 운영을 지원하고 주민 참여를 활성화할 수 있는 웹 플랫폼을 기획하게 되었습니다.

---

### 1.2. 개발 목표 및 주요 내용

본 프로젝트의 목표는 AI와 웹 플랫폼을 활용하여 작은도서관의 운영 효율을 높이고, 지역 주민과 대학생 봉사자가 함께 참여할 수 있는 지역 커뮤니티 기반 서비스를 구축하는 것입니다.

주요 목표는 다음과 같습니다.

1. **AI 기반 프로그램 기획 지원**
   - 주민이 제안한 지역 의제를 바탕으로 작은도서관 프로그램 기획안을 생성합니다.
   - 사서의 기획 업무 부담을 줄이고 다양한 문화·교육 프로그램 운영을 지원합니다.

2. **지역 주민 참여 활성화**
   - 주민이 우리 동네에 필요한 프로그램이나 해결해야 할 문제를 직접 제안할 수 있도록 합니다.
   - 제안된 의제를 AI가 분류·요약하여 사서가 주민 수요를 쉽게 파악할 수 있도록 합니다.

3. **대학생 봉사자 매칭**
   - 대학생의 전공, 관심 분야, 활동 가능 시간 등을 바탕으로 도서관 프로그램과 봉사자를 매칭합니다.
   - 전공 기반 멘토링 및 재능기부 활동을 통해 지역사회 연계를 강화합니다.

---

### 1.3. 세부내용

본 서비스의 주요 사용자는 **지역 주민**, **사서/관리자**, **대학생 봉사자**입니다.

#### 1) 지역 주민

지역 주민은 작은도서관에서 다루었으면 하는 지역 문제나 프로그램 아이디어를 제안할 수 있습니다.

주요 기능은 다음과 같습니다.

- 지역 의제 등록
- 도서관 프로그램 및 공지사항 확인
- 프로그램 참여 신청
- 다른 주민의 제안 확인
- 지역사회 문제 해결 과정 참여

#### 2) 사서 / 관리자

사서와 관리자는 주민이 등록한 의제를 확인하고, AI 분석 결과를 바탕으로 프로그램을 기획할 수 있습니다.

주요 기능은 다음과 같습니다.

- 주민 의제 확인 및 관리
- AI 기반 의제 분류 결과 확인
- AI 기반 프로그램 기획안 생성
- 프로그램 및 공지사항 등록·수정·삭제
- 대학생 봉사자 추천 결과 확인

#### 3) 대학생 봉사자

대학생 봉사자는 자신의 전공, 관심 분야, 활동 가능 시간 등을 등록하고 도서관 프로그램과 매칭될 수 있습니다.

주요 기능은 다음과 같습니다.

- 봉사자 정보 등록
- 관심 분야 및 활동 가능 시간 입력
- 도서관 프로그램 추천 확인
- 전공 기반 멘토링 및 재능기부 활동 참여

---

### 1.4. 기존 서비스 대비 차별성

기존 작은도서관 관련 서비스는 주로 도서관 검색, 기본 정보 제공, 행사 안내에 초점을 두고 있습니다. 반면 모이라는 작은도서관의 실제 운영을 지원하고, 주민·사서·대학생 봉사자를 연결하는 운영 지원형 플랫폼입니다.

| 구분 | 기존 작은도서관 서비스 | 모이라 |
| --- | --- | --- |
| 서비스 범위 | 전국 단위 정보 제공 중심 | 금정구 지역 밀착형 운영 지원 |
| 주요 기능 | 도서관 검색, 행사 안내, 운영 사례 제공 | AI 프로그램 기획, 지역 의제 수집, 봉사자 매칭 |
| 운영 지원 | 제한적 | 실제 운영 중심 지원 |
| 커뮤니티 기능 | 단순 정보 제공 중심 | 사서·주민·대학생 연결 |
| 봉사자 모집 | 일반 봉사활동 중심 | 전공 기반 멘토링 및 재능기부 특화 |
| 활용 대상 | 이용자 중심 | 사서, 주민, 대학생 봉사자까지 확대 |

모이라는 단순 정보 제공을 넘어 다음과 같은 차별성을 가집니다.

- AI 기반 지역 의제 분류 및 요약
- 주민 수요 기반 프로그램 기획 지원
- 금정구 작은도서관 중심의 지역 밀착형 서비스
- 대학생 전공 기반 봉사자 매칭
- 사서, 주민, 대학생이 함께 활용하는 통합 플랫폼

---

### 1.5. 사회적가치 도입 계획

모이라는 작은도서관을 중심으로 지역 주민, 사서, 대학생 봉사자가 함께 참여하는 지역 커뮤니티 플랫폼을 지향합니다.

본 프로젝트를 통해 다음과 같은 사회적 가치를 실현하고자 합니다.

1. **지역 주민 참여 확대**
   - 주민이 직접 지역 의제와 프로그램 아이디어를 제안할 수 있도록 지원합니다.
   - 지역 문제 해결 과정에 주민이 참여할 수 있는 구조를 마련합니다.

2. **작은도서관 운영 지원**
   - AI 기반 프로그램 기획 기능을 통해 사서의 업무 부담을 완화합니다.
   - 주민 수요를 반영한 문화·교육 프로그램 운영을 지원합니다.

3. **대학생 재능기부 활성화**
   - 대학생이 자신의 전공과 관심 분야를 활용해 지역사회에 기여할 수 있도록 합니다.
   - 아동·청소년 대상 교육 지원, 디지털 교육, 독서 멘토링 등 다양한 봉사 활동으로 확장할 수 있습니다.

4. **문화·정보 접근성 향상**
   - 주민이 가까운 작은도서관의 프로그램과 참여 기회를 쉽게 확인할 수 있도록 합니다.
   - 지역 내 문화·교육 접근성 격차 완화에 기여합니다.

5. **지역사회 협력 모델 구축**
   - 금정구 작은도서관 시범 운영을 시작으로 부산 지역 전체 작은도서관으로 확장하는 것을 목표로 합니다.
   - 지역 대학 및 기관과의 협력을 통해 지속 가능한 지역사회 참여 모델을 구축합니다.


## 2. 상세설계

### 2.1. 시스템 구성도

```mermaid
flowchart LR
    User[지역 주민  /  사서  /  대학생 봉사자] --> Frontend[Frontend<br/>Next.js + React]
    Frontend --> Backend[Backend<br/>NestJS]
    Backend --> DB[(PostgreSQL<br/>Prisma ORM)]
    Backend --> AI[AI API<br/>의제 분류  /  요약  /  기획안 생성]
    Backend --> Auth[인증 및 권한 관리]
```

시스템은 크게 프론트엔드, 백엔드, 데이터베이스, AI 기능 모듈로 구성됩니다.

#### Frontend

사용자 화면을 제공합니다.  
지역 의제 등록, 프로그램 조회, 로그인, 봉사자 등록 등의 UI를 담당합니다.

#### Backend

사용자 요청을 처리하고 데이터베이스와 AI 기능을 연동합니다.  
인증, 게시판, 프로그램, 의제, 봉사자 매칭 관련 API를 제공합니다.

#### Database

사용자 정보, 지역 의제, 프로그램 정보, 봉사자 정보, 게시글 데이터를 저장합니다.

#### AI 기능

지역 의제를 분류·요약합니다.  
선택된 의제를 바탕으로 도서관 프로그램 기획안 초안을 생성합니다.  
프로그램 조건과 봉사자 정보를 바탕으로 매칭을 지원합니다.

### 3.2. 기능설명

#### 1) 홈 화면

- 사용자는 서비스 소개와 주요 기능을 확인할 수 있습니다.
- 작은도서관 안내, 프로그램 정보, 공지사항, 지역 의제 요약 영역을 확인할 수 있습니다.
- 로그인, 회원가입, 지역 의제, 봉사자 연계 등 주요 페이지로 이동할 수 있습니다.

#### 2) 로그인 페이지

- 사서 및 관리자 계정은 이메일과 비밀번호를 입력하여 로그인할 수 있습니다.
- 입력값이 비어 있는 경우 로그인 버튼이 비활성화되거나 안내 문구가 표시됩니다.
- 유효성 검사를 통과하면 로그인 요청을 보낼 수 있는 데이터 구조를 생성합니다.
- 실제 인증 API 연동은 추후 구현 예정입니다.

#### 3) 지역 의제 등록 페이지

- 지역 주민은 우리 동네에 필요한 프로그램이나 해결해야 할 문제를 작성할 수 있습니다.
- 제목, 내용, 카테고리 등의 정보를 입력합니다.
- 등록된 의제는 관리자 화면에서 확인할 수 있습니다.

#### 4) AI 의제 분류 기능

- 주민이 등록한 의제를 AI가 분석합니다.
- 의제를 교육, 문화, 복지, 환경, 세대, 안전 등의 주제로 분류합니다.
- 핵심 키워드와 요약문을 생성하여 사서가 주민 요구를 빠르게 파악할 수 있도록 지원합니다.

#### 5) AI 프로그램 기획안 생성 기능

- 사서가 특정 지역 의제를 선택하면 AI가 프로그램 기획안 초안을 생성합니다.
- 기획안에는 다음 항목이 포함됩니다.
  - 프로그램명
  - 대상
  - 목적
  - 운영 방식
  - 준비물
  - 필요 봉사자
  - 기대 효과

#### 6) 대학생 봉사자 등록 및 매칭 기능

- 대학생 봉사자는 전공, 관심 분야, 활동 가능 시간 등을 입력합니다.
- 프로그램의 성격과 봉사자 정보를 비교하여 적합한 봉사자를 추천합니다.
- 사서 및 관리자는 추천된 봉사자 정보를 확인하고 프로그램 운영에 활용할 수 있습니다.

#### 7) 프로그램 및 게시판 관리

- 관리자는 도서관 프로그램과 공지사항을 등록, 수정, 삭제할 수 있습니다.
- 사용자는 프로그램 목록 및 상세 정보를 확인할 수 있습니다.
- 프로그램 상세 화면에서는 일정, 장소, 대상, 모집 인원, 신청 방법 등을 확인할 수 있습니다.

### 3.5. AI 도구 활용

- 홈 화면, 로그인 화면, 지역 의제 등록 화면, 관리자 화면 등의 UI 구조를 설계했습니다.
- 사용자가 이해하기 쉬운 문구와 입력 항목을 검토했습니다.

#### 3) 프론트엔드 / 백엔드 개발

- GitHub Copilot을 활용하여 반복적인 코드 작성과 컴포넌트 구조 생성을 보조했습니다.
- TypeScript 기반 코드 작성, 리팩토링, 오류 수정, 주석 작성 등에 활용했습니다.
- 프론트엔드에서는 Next.js와 React 기반 화면 컴포넌트 구현을 보조했습니다.
- 백엔드에서는 NestJS, Prisma 기반 API 및 데이터베이스 연동 코드 작성을 보조할 예정입니다.

#### 4) 테스트 데이터 생성

#### 5) 문서화 및 협업
